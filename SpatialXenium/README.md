# SpatialXenium Nextflow Pipeline

This Nextflow pipeline processes spatial omics data for analysis, starting from Xenium datasets and generating clustering, niche identification, and spatial visualization outputs.

## Aditional Information 

For a more detaild information of the pipeline and the modules see [`module_summary.md`](module_summary.md).

## Installation

Clone the repository:

```bash
git clone https://novonordiskit@dev.azure.com/novonordiskit/IntegratedOmics/_git/SpatialXenium
```

## Test Run

Execute a quick test run to ensure everything is set up correctly:

```bash
nextflow run SpatialXenium -profile test,marjorie
```

## Example Input

Prepare a sample sheet (e.g., `samplesheet.csv`) specifying your Xenium data paths:

```csv
sample,xenium_path
Sample_1,"/path/to/Sample_1_Xenium_data/"
Sample_2,"/path/to/Sample_2_Xenium_data/"
```

Use this file with the pipeline:

```bash
nextflow run SpatialXenium \
   -profile <docker/singularity/.../institute> \
   --input samplesheet.csv \
   --outdir <OUTDIR>
```

## Introduction

**nf-core/spatialxenium** is a bioinformatics pipeline designed to analyze spatial transcriptomics data generated by the 10x Genomics Xenium platform. It ingests spatially-resolved gene expression data, performs quality control, normalization, clustering, and spatial niche construction. The pipeline outputs processed RDS objects, cluster/niche compositions, top marker genes, and visualizations (PCA, UMAP, and spatial plots).

The pipeline is optimized to run on high-performance computing environments, ensuring efficient handling of large datasets.

### Optional Parameters

The following parameters can be adjusted to customize the pipeline's behavior. The numbers shown are the default values. The `reference_rds` parameter is optional; if not provided, label transfer will not be part of the analysis. By default, the column that will be transferred is `cell_type`.

- `min_reads_per_cell`: Minimum read count filter to remove low expressing cells. Default = 10
- `max_reads_per_cell`: Maximum read count filter to remove high expressing cells. Default = 1000000
- `max_control_prop_per_cell`: Maximum proportion of control probes or control codewords per cell. Default = 1.0
- `n_highly_var_genes`: Number of highly variable genes to select in the `FindVariableFeatures` Seurat function. Default = 1000
- `reference_rds`: Path to RDS object of single-cell RNASeq data to be used for cell label transfer. Default = "NONE"
- `single_cell_label_col`: Column of the single-cell RDS to use for label transfer. Default = "cell_type"
- `min_celltype_probability`: Minimum probability for cell type assignment during label transfer. Default = 0.5
- `genes`: Path to the file containing the list of genes to be used in the analysis. Default = "/path/to/genes.txt"
- `generic_name`: Use a generic name for the Xenium_Seurat_Object.rds file. Default = "FALSE" (use "TRUE" with quotes to activate.)
- `future_mem_limit`: Max memory (in bytes) for R's `future` package, e.g., 8e9 = 8GB. Default = 8e9

Example usage with custom parameters:

```bash
nextflow run nf-core/spatialxenium \
   -profile <docker/singularity/.../institute> \
   --input samplesheet.csv \
   --outdir <OUTDIR> \
   --min_reads_per_cell 20 \
   --max_reads_per_cell 500000 \
   --max_control_prop_per_cell 0.5 \
   --n_highly_var_genes 2000 \
   --reference_rds /path/to/reference.rds \
   --single_cell_label_col "custom_label"
   --genes /path/to/genes.txt
   --generic_name "TRUE"
   --future_mem_limit 9e9
```

## Pipeline Output

The pipeline generates several key components:

- **Processed Seurat Object**: The script generates a processed Seurat object for each sample, which includes normalized data, PCA and UMAP embeddings, and identified clusters.
- **QC Metrics**: Initial and final QC metrics are computed and saved, providing insights into the quality of the data before and after filtering.
- **Plots**: Various plots are generated to visualize QC metrics, PCA and UMAP embeddings, and gene expression patterns. These include scatter plots with marginal histograms, waterfall plots, and proportional bar plots.
- **Label Transfer Results**: If label transfer is performed, the script generates predictions for cell types and visualizes the predicted cell type frequencies and distributions.
- **Output Files**: The processed data, including the Seurat object, metadata, QC metrics, gene counts, and cell type predictions, are saved to the specified output directory.
